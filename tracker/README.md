# Grocery Freshness Tracker API

A Spring Boot REST API for tracking grocery items and their freshness/expiration status. The system provides full CRUD functionality for managing grocery inventory and is designed to help users reduce food waste by keeping track of expiration dates.

## Features

- Create, read, update, and delete grocery items
- Track expiration dates and item freshness status
- RESTful API design using Spring Boot
- In-memory data storage (currently)
- Clean separation of controller, service, and model layers

## Tech Stack

- Java
- Spring Boot
- Spring Web (REST API)
- Maven
- In-memory storage (temporary)

## Planned Improvements

- Migrate from in-memory storage to PostgreSQL for persistent data storage
- Add input validation and improved error handling
- Implement user authentication and multi-user support
- Build optional frontend dashboard
- Dockerize the application for easier deployment

## Project Goal

This project demonstrates backend development fundamentals using Spring Boot, focusing on REST API design, CRUD operations, and clean architecture. It serves as the foundation for a future full-stack grocery tracking application.
